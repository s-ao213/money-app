# 共同・個人家計簿アプリ 要件定義書

## 1. 概要

本アプリは、恋人同士など2人で一部のお金を共有しつつ、個人の収支もそれぞれ管理できる家計簿アプリである。

主な目的は以下のとおり。

- 2人で利用しているサブスクの契約者、負担割合、支払日、更新周期を整理する
- 今月それぞれが「誰に」「何に対して」「いくら」支払うべきかを明確にする
- 2人の間の貸し借りを、返済方法、返済期限、返済状況まで管理する
- 個人の収入・支出・仕事別収入源を月ごとに確認できるようにする
- 必要に応じてExcel形式で出力できるようにする
- SupabaseとVercelを用いて、セキュリティに配慮したWebアプリとして運用する

## 2. 想定ユーザー

### 2.1 利用者

- 利用者は基本的に2人
- それぞれが自分のアカウントでログインする
- 2人で共有するデータと、個人だけが見られるデータを明確に分ける

### 2.2 権限

- 自分の個人収支データは自分だけが閲覧・編集できる
- 共有サブスク、貸し借り、2人間の支払い予定は2人とも閲覧できる
- サブスクごとの最終的な負担割合・負担金額は、原則として契約者が決定できる
- ただし、相手が確認できるように変更履歴を残す

## 3. 技術構成

### 3.1 使用技術

- フロントエンド: React
- フレームワーク: Next.js
- バックエンド: Next.js API Routes または Server Actions
- データベース: Supabase PostgreSQL
- 認証: Supabase Auth
- ホスティング: Vercel
- Excel出力: xlsx系ライブラリ、またはサーバー側でExcelファイルを生成
- アイコン: Font Awesome、または既存UIに合うアイコンライブラリ

### 3.2 基本方針

- スマホとPCの両方で使いやすいレスポンシブWebアプリにする
- UIは白黒を基調にする
- 重要な確定・支払い・警告・削除などの操作には赤、青などのアクセントカラーを使う
- 家計簿アプリとして、装飾よりも一覧性、確認しやすさ、入力のしやすさを優先する

## 4. 管理するデータの分類

本アプリで扱うデータは大きく以下に分ける。

| 分類 | 共有範囲 | 内容 |
|---|---|---|
| 共有サブスク | 2人で共有 | 2人で使っているサブスク、契約者、支払い周期、負担割合など |
| 2人間の貸し借り | 2人で共有 | 借入、貸付、返済予定、返済実績など |
| 個人収入 | 本人のみ | 給料、副業、アルバイト、臨時収入など |
| 個人支出 | 本人のみ | 食費、交通費、個人サブスク、買い物など |
| 個人サブスク | 本人のみ | 自分だけで契約・利用しているサブスク |
| 月次集計 | 表示条件による | 今月の支払い予定、収支、返済状況など |

## 5. 機能要件

## 5.1 認証・アカウント機能

### 5.1.1 ユーザー登録

- メールアドレスとパスワードで登録できる
- 将来的にはGoogleログインなども追加可能にする
- 登録後、自分の表示名を設定できる

### 5.1.2 ログイン・ログアウト

- Supabase Authを利用してログインする
- ログアウトできる
- 未ログインの場合はログイン画面へ誘導する

### 5.1.3 ペア設定

- 2人を1つのペアとして紐づける
- 招待コード、招待リンク、または相手のメールアドレスによってペアを作成する
- ペアに紐づいていない場合は、共有サブスクや貸し借り機能は利用できない

### 5.1.4 ペア解除

- ペアを解除できる
- 解除時には、共有データの扱いを選べるようにする
  - 自分のコピーとして保持する
  - 閲覧のみ可能にする
  - 完全削除する
- 初期版では「解除は管理画面からのみ」など、安全側の仕様にしてもよい

## 5.2 ダッシュボード機能

### 5.2.1 月次ダッシュボード

ログイン後、今月の状況が一目でわかる画面を表示する。

表示内容は以下。

- 今月、自分が支払う予定の合計額
- 今月、相手に支払う予定の合計額
- 今月、サブスク会社など外部に支払う予定の合計額
- 今月、相手から受け取る予定の合計額
- 今月の個人収入合計
- 今月の個人支出合計
- 今月の収支差額
- 返済期限が近い貸し借り
- 支払日が近いサブスク

### 5.2.2 月の切り替え

- 年月を選択して過去・未来の予定を確認できる
- デフォルトは現在月
- 例: 2026年7月、2026年8月など

### 5.2.3 支払い先の分類

支払い予定は以下のように分けて表示する。

- 相手に払うもの
- 相手から受け取るもの
- サブスクサービスに直接払うもの
- 個人支出として払うもの
- 貸し借りの返済として払うもの

## 5.3 共有サブスク管理機能

### 5.3.1 サブスク登録

共有サブスクとして以下を登録できる。

- サブスク名
- 契約者
- 利用者
- 月額または年額の料金
- 支払い周期
  - 毎月
  - 年1回
  - 任意周期
- 支払日
- 次回更新日
- 支払い方法
  - クレジットカード
  - 銀行引き落とし
  - 電子マネー
  - その他
- 負担方式
  - 比率指定
  - 固定金額指定
  - 全額どちらかが負担
- 自分の負担額
- 相手の負担額
- メモ
- 有効・停止状態

### 5.3.2 契約者と支払いの考え方

- 契約者はサブスクサービスに対して全額を支払う
- 契約者ではない相手は、決められた比率または金額に従って契約者へ支払う
- そのため、アプリ上では以下を別々の支払いとして扱う
  - 契約者がサブスクサービスへ払う支払い
  - 相手が契約者へ払う精算支払い

### 5.3.3 負担方式

#### 比率指定

例:

- サブスク料金: 2,000円
- 契約者: A
- Aの負担割合: 50%
- Bの負担割合: 50%
- Aはサブスクサービスへ2,000円支払う
- BはAへ1,000円支払う

#### 固定金額指定

例:

- サブスク料金: 1,980円
- 契約者: A
- Bの負担額: 800円
- Aはサブスクサービスへ1,980円支払う
- BはAへ800円支払う
- Aの実質負担は1,180円

#### 片方全額負担

例:

- 契約者Aが全額負担
- BからAへの精算支払いは発生しない

### 5.3.4 支払い周期

#### 毎月払い

- 毎月指定日に支払い予定を作成する
- 支払日が31日の場合、31日が存在しない月は月末に補正する

#### 年1回払い

- 年に1回、更新月・更新日に支払い予定を作成する
- 年額をその月にまとめて支払う形式を基本とする
- 画面上では参考として月割り金額も表示できる

#### 任意周期

初期版では必須ではないが、将来的に以下を扱えるようにする。

- 3か月ごと
- 6か月ごと
- 2年ごと
- 不定期

### 5.3.5 サブスク一覧

一覧画面では以下を表示する。

- サブスク名
- 契約者
- 金額
- 支払い周期
- 次回支払日
- 自分の負担額
- 相手の負担額
- 今月支払いあり・なし
- 有効・停止状態

### 5.3.6 サブスク詳細

詳細画面では以下を表示する。

- 基本情報
- 負担設定
- 今後の支払い予定
- 過去の支払い履歴
- 変更履歴
- メモ

### 5.3.7 支払い完了管理

- サブスクサービスへの支払いを「支払い済み」にできる
- 相手への精算支払いを「支払い済み」にできる
- 支払い済み日を記録する
- 支払い方法を記録できる
- 未払い、支払い済み、遅延などの状態を管理する

## 5.4 2人間の貸し借り管理機能

### 5.4.1 貸し借り登録

貸し借りとして以下を登録できる。

- 取引名
- 貸した人
- 借りた人
- 元金
- 発生日
- 返済期限
- 返済方法
  - 一括
  - 分割
  - 任意返済
- 分割回数
- 月ごとの返済予定額
- 返済開始月
- 返済日
- メモ
- ステータス
  - 返済中
  - 完済
  - 期限超過
  - 取り消し

### 5.4.2 一括返済

- 指定した返済期限までに全額返済する
- 今月が返済月の場合、ダッシュボードに表示する
- 返済済み金額と残額を表示する

### 5.4.3 分割返済

例:

- 借入額: 60,000円
- 分割回数: 6回
- 月ごとの予定返済額: 10,000円
- 返済日: 毎月25日

表示内容:

- 今月返済すべき金額
- 今月実際に返済した金額
- 返済予定に対して不足している金額
- 累計返済額
- 残額
- 完済予定日

### 5.4.4 任意返済

- 毎月の固定返済額は設定しない
- 返済があったときに金額を登録する
- 残額と期限だけを管理する

### 5.4.5 返済実績登録

返済実績として以下を登録できる。

- 対象の貸し借り
- 返済日
- 返済額
- 支払い方法
  - 現金
  - 銀行振込
  - PayPayなどの送金
  - その他
- メモ

### 5.4.6 取引ごとの表示

各貸し借り取引では以下が確認できる。

- 元金
- 返済済み合計
- 残額
- 今月返すべき金額
- 今月返した金額
- 返済期限
- 返済進捗率
- 返済履歴
- 次回返済予定

## 5.5 個人家計簿機能

### 5.5.1 個人収入登録

個人の収入として以下を登録できる。

- 収入名
- 金額
- 入金日
- 収入源
  - 本業
  - 副業
  - アルバイト
  - フリーランス
  - 臨時収入
  - その他
- 継続収入か単発収入か
- メモ

### 5.5.2 収入源管理

- 複数の仕事・収入源を登録できる
- 収入源ごとに月別合計を確認できる
- 収入源ごとの年間合計も確認できる

### 5.5.3 個人支出登録

個人の支出として以下を登録できる。

- 支出名
- 金額
- 支出日
- カテゴリ
  - 食費
  - 日用品
  - 交通費
  - 医療
  - 美容
  - 趣味
  - 交際費
  - 仕事関連
  - 個人サブスク
  - その他
- 支払い方法
- メモ

### 5.5.4 個人サブスク

- 共有ではない、自分だけのサブスクを登録できる
- 個人サブスクは相手には見えない
- 月次収支には自動で支出として反映される

### 5.5.5 貸し借り・サブスクとの連携

個人収支には以下も反映できる。

- 相手から返済を受けた金額
- 相手へ返済した金額
- 相手から受け取ったサブスク精算金
- 相手へ支払ったサブスク精算金
- 自分が契約者としてサブスクサービスに支払った金額
- 個人サブスクの支払い

ただし、集計時には二重計上を避ける必要がある。

例:

- 契約者Aが共有サブスク2,000円を支払う
- BがAへ1,000円支払う
- Aの個人収支では、外部支出2,000円とBからの精算収入1,000円を表示できる
- Aの実質負担は1,000円として表示する

## 5.6 月次集計機能

### 5.6.1 個人月次集計

月ごとに以下を表示する。

- 収入合計
- 支出合計
- 収支差額
- 収入源別収入
- カテゴリ別支出
- 個人サブスク合計
- 共有サブスクの実質負担額
- 借金返済額
- 貸付回収額
- 未払い予定額

### 5.6.2 共有月次集計

2人の共有情報として以下を表示する。

- 共有サブスク総額
- Aが契約者として外部へ支払う金額
- Bが契約者として外部へ支払う金額
- AからBへの精算予定額
- BからAへの精算予定額
- AからBへの返済予定額
- BからAへの返済予定額
- 今月の未払い一覧

### 5.6.3 支払い予定一覧

支払い予定は日付順に表示する。

表示項目:

- 支払日
- 支払い種別
  - 共有サブスク外部支払い
  - 共有サブスク精算
  - 借金返済
  - 個人支出
  - 個人サブスク
- 支払う人
- 受け取る人、または支払い先
- 金額
- ステータス

## 8. セキュリティ要件

## 8.1 認証

- Supabase Authを利用する
- パスワードはSupabase側で安全にハッシュ化される
- アプリ側で平文パスワードを保存しない
- セッション管理はSupabaseの仕組みに従う

## 8.2 データアクセス制御

- Supabase Row Level Securityを必ず有効にする
- 個人データは本人のみアクセス可能にする
- 共有データは同じペアに所属している2人のみアクセス可能にする
- ペアに属していない第三者は共有データを取得できないようにする

## 8.3 ハッシュ化と暗号化の考え方

ユーザー要望として「Supabaseなどもハッシュ値などを用いてわからないようにしたい」とあるが、家計簿データには以下の整理が必要である。

### ハッシュ化に向いているもの

- パスワード
- 招待コードの照合値
- 復元する必要のない識別情報

ハッシュ化したデータは基本的に元に戻せないため、金額やメモのように画面で表示する必要がある情報には向かない。

### 暗号化に向いているもの

- 金額
- メモ
- 支払い方法の補足
- 個人的な収支内容

画面に表示する必要があるデータは、必要に応じて暗号化を検討する。

初期版では以下を推奨する。

- Supabase RLSでアクセス制御を厳密に行う
- 機密性の高いメモ欄は暗号化対象にする
- 招待コードはハッシュ化して保存する
- パスワードはSupabase Authに任せる
- クレジットカード番号などの決済情報は保存しない

## 8.4 保存しない情報

以下は原則として保存しない。

- クレジットカード番号
- 銀行口座番号
- マイナンバー
- 本人確認書類
- 外部サービスのログインパスワード

## 8.5 監査ログ

以下の操作は履歴を残す。

- 共有サブスクの作成・変更・削除
- 負担割合・負担金額の変更
- 貸し借りの作成・変更・削除
- 返済実績の登録・削除
- 支払い済み処理

## 8.6 削除

- 誤操作防止のため、重要データは論理削除を基本とする
- 完全削除は設定画面または管理処理として扱う

## 9. 通知・アラート要件

初期版ではアプリ内通知を優先し、将来的にメール通知などを追加する。

### 9.1 アプリ内アラート

- サブスク支払日が近い
- サブスク精算が未払い
- 返済期限が近い
- 返済予定額に対して実績が不足している
- 期限超過がある

## 10. 非機能要件

## 10.1 パフォーマンス

- ダッシュボードは素早く表示されること
- 月次集計は通常利用で数秒以内に表示されること
- 大量データがある場合もページネーションや絞り込みで操作しやすくする

## 10.2 可用性

- VercelとSupabaseの標準的な可用性に依存する
- 重要なデータは定期的にエクスポートできるようにする

## 10.3 保守性

- 共有サブスク、貸し借り、個人収支を機能単位で分けて実装する
- 金額計算ロジックは共通関数として管理する
- 支払い予定生成ロジックはテストしやすい形にする

## 10.4 拡張性

## 11. 計算ルール

## 11.1 金額の丸め

- 日本円を前提とし、小数点は扱わない
- 比率計算で端数が出た場合の扱いを決める必要がある
- 初期案では、契約者以外の負担額を四捨五入し、差額は契約者側の負担にする

## 11.2 年額サブスク

年額サブスクは以下の2種類の表示を行う。

- 実際の支払い月に年額を表示
- 参考情報として月割り金額を表示

月次の「実際に今月払う金額」には、支払い月だけ年額を含める。

## 11.3 支払い日が存在しない月

- 29日、30日、31日が存在しない月は月末を支払日とする

## 11.4 返済不足

分割返済で予定より少ない金額しか返済されていない場合は、不足額を表示する。

例:

- 今月の予定返済額: 10,000円
- 今月の実績返済額: 6,000円
- 今月の不足額: 4,000円

不足額を翌月に自動で繰り越すかどうかは設定できるようにする。

初期案では、未返済額として残額に含め、翌月の予定額とは別に不足額として表示する。

## 13. 未確定事項・確認したいこと

今後、実装前に確認したい点は以下。

1. 利用者は完全に2人固定でよいか、将来的に3人以上も想定するか
2. サブスクの負担割合は、契約者だけが編集できる仕様でよいか
3. 相手が負担割合に同意したことを記録する承認機能が必要か
4. 返済不足がある場合、翌月の予定額に自動加算するか、別表示にするか
5. 年額サブスクは実支払い月だけ支出に入れるか、月割りでも家計簿に反映するか
6. 個人収支に、相手との精算を自動反映するか、手動で反映するか
7. Excel出力は個人ごとのみか、共有データを含む総合レポートも必要か
8. 削除したデータを復元できるようにするか
9. メモ欄の暗号化を初期版から実装するか
10. スマホ利用を最優先にするか、PCでの表管理を最優先にするか

## 14. 初期リリース時の成功条件

初期リリース時点で、以下ができれば成功とする。

- 2人がそれぞれログインできる
- 2人がペアとして紐づく
- 共有サブスクを登録し、今月どちらがいくら払うべきか確認できる
- 契約者が外部サービスへ払う金額と、相手が契約者へ払う金額を分けて確認できる
- 貸し借りを登録し、返済予定・返済済み・残額を確認できる
- 個人の収入と支出を月別に確認できる
- Excelに出力できる
- 相手の個人家計簿データは見えない
- Supabase RLSにより、他人のデータへアクセスできない
